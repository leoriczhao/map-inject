整体架构

UjAPI 是一个运行时注入型 JASS 扩展框架，通过独立的加载器（UjAPIApp.exe）在 Warcraft III 启动时将 UjAPI.dll 注入到游戏进程中，hook JASS 虚拟机，注册数千个新的 native 函数。

注入流程

UjAPI 的注入方式与 YDWE 内置 JAPI 类似，但更独立。它通过 UjAPIApp.exe（.NET 加载器）启动游戏并注入 DLL。

方式一：UjAPIApp.exe 加载器（官方方式）

UjAPIApp.exe (.NET 4.7.2) │ ├─ 1. 用户配置 war3.exe 路径 ├─ 2. CreateProcess(war3.exe) → 挂起态启动 ├─ 3. VirtualAllocEx → 在目标进程分配内存 ├─ 4. WriteProcessMemory → 写入 UjAPI.dll 路径 ├─ 5. CreateRemoteThread → 调用 LoadLibrary 加载 UjAPI.dll │ └─ UjAPI.dll DllMain ├─ 6. Hook game.dll 的 nfunction_add 函数 ├─ 7. 拦截所有后续 native 注册 ├─ 8. 注入 2333 个扩展 native └─ 9. Hook 事件系统、伤害系统、Frame 系统等

方式二：MPQ 内嵌（callback 模式）

UjAPI.dll 也可以打包进地图 MPQ，通过 callback exploit 的 DLL 加载链自动加载。此方式在 1.27a 上完全可用，callback 中有完整的版本检测逻辑：

// callback::main() 中的版本检测
set i = ReadRealMemory(GetBytecodeAddress())
set i = i - ReadRealMemory(i)
if i == 2586768 then       → newInit27a()  // 1.27a ✓
elseif i == 5205600 then   → newInit26()   // 1.26a ✓
elseif i == 5276928 then   → newInit24b()  // 1.24b ✓
elseif i == 5276840 then   → newInit24e()  // 1.24e ✓
endif

// newUnlockMemory() 中额外支持：
if a == 2889044 then       → newInit27B()  // 1.27b ✓
elseif a == 2894996 then   → newInit28a()  // 1.28a ✓

每个版本有独立的偏移量表（pJassEnvAddress、pVirtualAlloc、pExportFromMpq 等），1.27a 是优先级最高、支持最完整的版本。

内存利用机制

UjAPI 的核心能力建立在 JASS 虚拟机的类型混淆漏洞之上。这是从 YDWE 时代继承的 exploit，通过 jasshelper 的 l__ 前缀变量实现。

类型混淆原理

JASS 是强类型语言，但 jasshelper（vJASS 编译器）在处理 l__ 前缀的局部变量时存在类型混淆：

// JASS 中，integer 和 code 是不同类型
local integer l__Code    // jasshelper 将 l__Code 视为 integer
local code    l__Int     // jasshelper 将 l__Int 视为 code

// 但实际编译后，它们共享同一个栈槽位
// 通过巧妙的函数调用，可以实现类型欺骗：
function setCode takes code c returns nothing
    set l__Code = c
    return  // 阻止 jasshelper 内联
endfunction

function C2I takes code c returns integer
    call setCode(c)       // 写入 l__Code（code 类型）
    loop
        return l__Code    // 读取 l__Code（但此时被当作 integer）
    endloop
    return 0
endfunction
// 结果：code → integer 的任意转换

内存读写实现

通过 Memory[] 数组实现任意地址读写：

// Memory 是一个 integer array，其数据区在进程堆中
// 通过 bytecode 操纵，可以将 Memory 的基址映射到任意地址

// 步骤 1：通过 JASS bytecode 创建一个新数组
// 步骤 2：利用 SetUnitUserData 将句柄表地址映射到数组索引
// 步骤 3：修改数组的内部指针，使 Memory[] 指向任意地址

function ReadRealMemory takes integer addr returns integer
    if addr / 4 * 4 != addr then
        return ReadRealMemory_FIX(addr)  // 非对齐读取
    endif
    return Memory[addr / 4]             // 直接读取
endfunction

function WriteRealMemory takes integer addr, integer val returns nothing
    set Memory[addr / 4] = val           // 直接写入
endfunction

可执行内存分配

// 通过 hook MergeUnits native 调用 VirtualAlloc
function AllocateExecutableMemory takes integer size returns integer
    // hook MergeUnits → VirtualAlloc(addr, size, MEM_COMMIT, PAGE_EXECUTE_READWRITE)
    set retval = B2I(MergeUnits(0, size + 4, 0x3000, 0x40))
    // 恢复 MergeUnits 的原始地址
    return (retval + 4) / 4 * 4
endfunction

Native 调用 Trampoline

// 在分配的可执行内存中构造 x86 shellcode
// 支持 stdcall / cdecl / fastcall 三种调用约定

function CallStdcallWith1Args takes integer pFunc, integer arg1 returns integer
    // 写入 shellcode：
    // push ecx; mov ecx,ecx
    // push arg1
    // mov ecx, pFunc
    // call ecx; pop ecx; ret
    call WriteRealMemory(pReservedExecutableMemory2 + 0,  0x68C98B51)
    call WriteRealMemory(pReservedExecutableMemory2 + 4,  arg1)
    call WriteRealMemory(pReservedExecutableMemory2 + 8,  0xB990C98B)
    call WriteRealMemory(pReservedExecutableMemory2 + 12, pFunc)
    call WriteRealMemory(pReservedExecutableMemory2 + 16, 0xC359D1FF)
    // 通过 hook IgnoredUnits native 执行 shellcode
    set pOffset1 = IgnoredUnits(0)
    return pOffset1
endfunction

Native 调度机制

UjAPI 的 native 函数通过 hashtable dispatch 模式从 JASS 侧调用 C++ 实现。

调度流程

JASS 调用侧： ┌─────────────────────────────────────────────────────────────────┐ │ function EXGetUnitAbility takes unit u, integer abilId returns integer │ │ call SaveInteger(japi_ht, japi__key, 1, GetHandleId(u)) │ │ call SaveInteger(japi_ht, japi__key, 2, abilId) │ │ call SaveStr(japi_ht, japi__key, 0, "(II)I") ← 签名字符串 │ │ call UnitId("EXGetUnitAbility") ← 函数名 │ │ return LoadInteger(japi_ht, japi__key, 0) ← 返回值 │ │ endfunction │ └─────────────────────────────────────────────────────────────────┘ │ ▼ C++ 侧（UjAPI.dll 中）： ┌─────────────────────────────────────────────────────────────────┐ │ 1. UnitId("EXGetUnitAbility") 被 hook │ │ 2. 从 hashtable 读取参数：key=1,2,3... │ │ 3. 解析签名 "(II)I" → 2个integer参数, 返回integer │ │ 4. 查找 native 注册表 → 调用实际 C++ 函数 │ │ 5. 将返回值写回 hashtable key=0 │ └─────────────────────────────────────────────────────────────────┘

签名编码

字符含义说明
IInteger32 位整数
RReal浮点数（32 位）
BBoolean布尔值
SString字符串指针
HHandle句柄 ID
VVoid无返回值

示例：(II)R = 两个 integer 参数，返回 real。(IIIS)V = 三个 integer + 一个 string，无返回值。
