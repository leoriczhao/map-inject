🧩 jass.* 模块详解

japi.dll 提供 6 个 Lua 预加载模块，通过 luaL_requiref 注册（lua_bootstrap.cpp:309-325），对齐内置 JAPI 的五模块结构。

1. jass.common lua_bootstrap.cpp

JASS native 调用的核心模块。用户通过 require "jass.common" 获得 jass 表，所有 1167 个 JASS natives 通过 metatable __index 自动映射。

API 速查

jass.NativeName(args)
直接调用任意 JASS native（metatable __index）
jass.call(name, ...)
显式调用 JASS native（jass_call 函数）

实现原理

  1. 创建 jass 全局 table（luaopen_jass_common, L263）
  2. 设置 metatable __index → jass_index 函数
  3. jass_index（L191）：接收属性名 → 查 g_native_map → 返回 wrapper C closure
  4. Wrapper closure 持有 native_name 和 NativeInfo* 作为 upvalue
  5. 调用时 build_params（L83）解析参数字符串（如 "(Hplayer;RRRS)V"）并转换 Lua 值 → JASS 类型
  6. jass::call（jass_call.cpp）执行实际的 x86 调用
  7. push_retval 将 JASS 返回值推回 Lua 栈

参数字符串语法

格式示例说明
(II)I两个 int，返回 intI=integer, R=real, S=string, B=boolean, H=handle, C=code, V=void
(Hplayer;RRRS)Vhandle, 3 real, string → void分号分隔，小写后缀为描述（忽略）
(S)S一个 string，返回 string字符串通过 string_fake 传递

Lua 使用示例

local jass = require "jass.common"

-- 方式 1: 直接属性访问（metatable __index）
local player = jass.GetLocalPlayer()
jass.DisplayTimedTextToPlayer(player, 0, 0, 10, "Hello from Lua!")

-- 方式 2: 显式调用
local p2 = jass.call("GetLocalPlayer")
jass.call("DisplayTimedTextToPlayer", player, 0, 0, 10, "Via call()")

3. jass.japi lua_japi.cpp

运行时 JASS native 注册模块。允许 Lua 侧动态注册新的 JASS native 函数，这些 native 在 JASS 代码中可直接调用。

API 速查

japi("name", "def", fn)
注册新 JASS native（__call 元方法）
japi.NativeName(args)
调用已注册 native（__index 元方法）
pairs(japi)
遍历所有已注册 native（__pairs 元方法）

实现原理（三层架构）

第 1 层：x86 Thunk（14 字节）

每当 japi("name", "def", fn) 被调用时（lua_japi.cpp:366），生成一段 14 字节的可执行代码：

68 xx xx xx xx    push <native_id>      ; 标识这个 native
E8 xx xx xx xx    call <dispatcher>      ; 跳转到统一分发器
83 C4 04          add esp, 4             ; 清理 push
C3                ret                    ; 返回 JASS VM

Thunk 通过 ThunkAllocator（L51）分配在 VirtualAlloc(PAGE_EXECUTE_READWRITE) 的 RWX 页上。

第 2 层：Naked Dispatcher（x86 汇编）

japi_dispatcher（L189）是裸函数：

  1. 保存 ebp, ecx, edx
  2. 从栈上读取 native_id 和 jass_param[] 指针
  3. 调用 C++ 函数 japi_dispatch_impl(native_id, params)
  4. 返回后恢复寄存器，ret 回到 JASS VM

第 3 层：C++ Dispatch Impl

japi_dispatch_impl（L90）：

  1. 从 g_japi_by_id 查找 native 信息
  2. 从 LUA_REGISTRYINDEX 取出 Lua 回调函数
  3. 根据参数类型（INTEGER/REAL/STRING/BOOLEAN/HANDLE/CODE）将 JASS 参数推入 Lua 栈
  4. lua_pcall(L, nparams, 1, 0) 执行 Lua 回调
  5. 根据返回类型读取返回值并返回给 JASS

Lua 使用示例

local japi = require "jass.japi"

-- 注册一个新 native：TestJapiAdd(II)I
japi("TestJapiAdd", "(II)I", function(a, b)
    return a + b
end)

-- JASS 侧现在可以调用: call TestJapiAdd(10, 32) → 42
-- Lua 侧也可以直接调用:
local result = japi.TestJapiAdd(10, 32)  -- 42

-- 遍历已注册 native
for _, v in pairs(japi) do
    print(v.name, v.def, v.thunk)
end

关键源文件

4. jass.message lua_message.cpp ~857 行，最大模块

输入系统模块。通过 hook CGameUI 的 vtable 拦截键盘/鼠标消息，并提供世界坐标转换、选中单位查询、技能栏查询和命令发布功能。

API 速查

message.mouse() → wx, wy
当前鼠标世界坐标（优先缓存，fallback convert_xy）
message.selection() → handle
当前选中单位 handle（或 nil）
message.button(x, y) → id, order, target_type
技能栏按钮查询（x:0-3, y:0-2）
message.keyboard[code]
键盘状态表（整数键码 → true/false）
message.hook = fn
设置/清除消息回调
message.order_immediate(order, flags)
无目标命令
message.order_point(order, x, y, flags)
点目标命令
message.order_target(order, x, y, target, flags)
单位目标命令
message.order_enable_debug(en)
启用/禁用命令调试日志（true=开启, false=关闭）

消息类型枚举

常量值说明
MSG_KEY_DOWN0x40060064键盘按下
MSG_KEY_UP0x40060066键盘释放
MSG_MOUSE_DOWN0x1A0068鼠标按下（含世界坐标+屏幕坐标）
MSG_MOUSE_UP0x400500C9鼠标释放（无坐标数据）
MSG_ABILITY_MOUSE0x30064技能鼠标事件

Hook 消息格式

-- 键盘事件
{ type="key_down"|"key_up", category="keyboard", code=keycode, keystate=state, msgid=raw_id }

-- 鼠标事件
{ type="mouse_down", category="mouse", wx=world_x, wy=world_y, rx=screen_x, ry=screen_y, code=button, msgid=raw_id }
{ type="mouse_up",   category="mouse", msgid=raw_id }
{ type="mouse_ability", category="ability", msgid=raw_id }

-- 回调签名
message.hook = function(msg)
    -- msg.type, msg.category, msg.code, msg.wx, msg.wy, ...
    return true   -- true = 继续传递事件, false = 拦截
end

实现原理

  1. CGameUI vtable hook（init_cgameui_hook, L816）：通过 RTTI 字符串 ".?AVCGameUI@@" 定位 CGameUI vtable，用 replace_pointer 将 vtable[4] 替换为 cgameui_handler
  2. 消息分发（cgameui_filter, L498）：解析消息结构体，提取 keycode/坐标/按钮数据，组装 Lua table，调用用户回调
  3. convert_xy（L386）：通过 CWorldFrameWar3 vtable[14] 定位坐标转换函数，将屏幕坐标转为世界坐标
  4. 对象→handle 逆查（object_to_handle, L255）：遍历 JASS VM 的 handle_table reverse_table，通过 object 指针反查 handle 值
  5. 命令调度（L710-746）：从 game.dll + 0x3AE4E0 扫描 4 个 CALL 指令，分别定位 immediate_order / point_order / target_order 函数

Lua 使用示例

local message = require "jass.message"

-- 键盘 hook
message.hook = function(msg)
    if msg.type == "key_down" then
        print(string.format("Key pressed: 0x%02X state=%d", msg.code, msg.keystate))
        if msg.code == 0x1B then  -- ESC
            message.hook = nil
        end
    end
    return true
end

-- 查询当前鼠标世界坐标
local wx, wy = message.mouse()

-- 查询选中单位
local unit_handle = message.selection()

-- 技能栏按钮（右上角按钮 = x=3, y=0）
local ability_id, order_id, target_type = message.button(3, 0)

-- 发布命令
message.order_point(0xD000F, 0, 0, 0)  -- smart 到原点

6. jass.hook lua_hook.cpp

JASS native 钩子模块。允许 Lua 拦截任意 JASS native 调用，修改参数、返回值或完全替换行为。

API 速查

hook("name", callback)
安装钩子（__call 元方法）
hook.unhook("name")
移除钩子

回调签名

hook("NativeName", function(param1, param2, ..., orig)
    -- orig 是原始 native 函数的引用，可调用以执行原始行为
    local result = orig(param1, param2, ...)
    return modified_result
end)

实现原理

  1. Thunk 替代：与 japi 模块共用 14 字节 x86 thunk 布局（ThunkCode, L32）
  2. 安装 hook（hook_call, L252）：(a) 解析参数字符串；(b) 存储回调到 LUA_REGISTRYINDEX；(c) 分配 thunk 跳转到 hook_dispatcher；(d) 保存原始 NativeInfo；(e) 直接修改 g_native_map 中该 native 的 address 字段为 thunk 地址
  3. Hook Dispatcher（hook_dispatch_impl, L105）：JASS 参数 → Lua 栈 → lua_pcall → 返回值 → JASS。与 japi_dispatcher 类似，但额外传递 orig 函数
  4. orig 函数：wrapper C closure 持有原始 NativeInfo.address 和 param spec，调用时走原始路径
  5. 卸载 hook（hook_unhook, L324）：恢复 NativeInfo.address 为原始值

与 jass.japi 的关键区别

维度jass.japijass.hook
用途注册新 JASS native拦截已有 JASS native
注册方式nfunction_add修改 NativeInfo.address
是否有 orig无有（可调用原始实现）
卸载方式不支持卸载hook.unhook() 恢复原始地址

Lua 使用示例

local hook = require "jass.hook"

-- 拦截 TestJapiAdd，将结果乘以 2
hook("TestJapiAdd", function(a, b, orig)
    return orig(a, b) * 2
end)
assert(japi.TestJapiAdd(5, 7) == 24)  -- 5+7=12, *2=24

-- 卸载 hook
hook.unhook("TestJapiAdd")
assert(japi.TestJapiAdd(5, 7) == 12)  -- 恢复正常

关键源文件

lua_hook.cppopen_hook(L349), hook_call(L252), hook_unhook(L324), hook_dispatcher(L230), hook_dispatch_impl(L105), HookInfo(L16), SimpleThunkAlloc(L42)
